# Darwin Gödel Machine: Open-Ended Evolution of Self-Improving Agents

Jenny Zhang, Shengran Hu, Cong Lu, Robert Lange, Jeff Clune

University of British Columbia, Vector Institute, Sakana AI

## Abstract

Most contemporary AI systems operate within fixed, human-designed architectures and cannot autonomously improve themselves. We introduce the Darwin Gödel Machine (DGM), a self-referential system that iteratively modifies its own code to enhance coding capabilities. Rather than requiring formal proofs of beneficial changes (as theoretical Gödel machines demand), the DGM empirically validates modifications using coding benchmarks. The system maintains an archive of generated agents and samples from it for self-modification, inspired by Darwinian evolution and open-endedness principles. Results demonstrate performance improvements from 20.0% to 50.0% on SWE-bench and 14.2% to 30.7% on Polyglot, significantly outperforming baselines lacking self-improvement or open-ended exploration.

## Introduction

Scientific progress operates cumulatively, with innovations building on prior discoveries. Current AI systems, despite their sophistication, remain constrained by fixed architectures designed by humans. Unlike the scientific method, which enables continuous self-directed advancement, today's AI development still depends heavily on human intervention. This paper investigates automating the search for progressively better AI systems.

Schmidhuber's theoretical Gödel machines represent one approach, relying on formal proofs ensuring modifications improve performance. However, proving that changes benefit complex systems like large language models is practically impossible. The actual impact of modifications depends heavily on context---a testing tool optimized for one environment might confuse an agent in another.

The Darwin Gödel Machine relaxes this requirement, instead using empirical validation through benchmarking. This mirrors biological evolution, where mutations aren't verified in advance but produced, tested, and selected through natural selection. The DGM combines this with inspiration from Darwinian principles and open-endedness research, maintaining a library of discovered agents as stepping stones for future generations.

## Darwin Gödel Machine

A Gödel Machine theoretically searches for provably beneficial self-improvements. The DGM relaxes this impractical requirement, instead requiring empirical evidence from experiments. This empiricism addresses the impossibility of proving changes improve complex systems.

Since empirical evidence can trap systems in local optima within vast algorithmic search spaces, the DGM maintains an archive of discovered solutions, enabling open-ended exploration rather than single-solution evolution. This mirrors Darwinian evolution, where innovations emerge by selecting entities from previous discoveries, modifying them, and retaining interesting variations.

### Self-Referential Self-Improvement of Coding Agents

The DGM initializes with one coding agent and evaluates progress on coding benchmarks. A coding agent represents a system implemented as a code repository, powered by frozen pretrained foundation models, capable of reading, writing, and executing code.

Self-improvement is defined as modifying the agent's own codebase components---tools, workflows, prompts. Python's Turing-completeness enables building any computable machine. The key motivation: empirical improvements must reflect both downstream task-solving ability and self-modification capacity. By configuring the DGM as a coding agent and testing coding capabilities, observed improvements demonstrate enhanced problem-solving and future self-improvement potential.

### Population-Based Open-Ended Exploration

Starting from a single agent, the DGM accumulates all generated variants in an archive. Each iteration selects parent agents for self-modification, roughly proportional to performance and inversely proportional to the number of their children. This balances exploitation of strong performers with exploration of promising, undersampled lineages.

All agents retain non-zero selection probability, ensuring any improvement pathway remains feasible. Each selected parent analyzes benchmark evaluation logs, proposes the next feature, and executes the modification. Newly generated agents undergo quantitative benchmark evaluation. Only agents compiling successfully and retaining codebase-editing ability join the archive; others are discarded.

Crucially, archived solutions can later serve as stepping stones for improvements discovered much later---substantially differentiating this from hill-climbing approaches.

## Experiments

### Setup

The DGM initializes with a lightweight coding agent built around a frozen foundation model with tool-use capabilities. The base agent accesses two tools: a Bash tool for command execution and an edit tool for viewing and modifying files. The intentionally lightweight design provides a simple starting point demonstrating clear improvement.

The DGM ran for 80 iterations, generating one new agent per iteration, with parallelization. During self-modifications, agents used Claude 3.5 Sonnet.

### Results

After 80 iterations, coding agent performance increased from 20.0% to 50.0% on SWE-bench and from 14.0% to 38.0% on Polyglot. DGM-discovered agents achieve performance comparable to or exceeding handcrafted solutions.

The DGM automatically enhanced both tools and foundation model workflows. Tool improvements included granular file viewing and precise editing. Workflow improvements encompassed multiple solution attempts and using another FM to evaluate and select optimal solutions.

Open-ended exploration allows escape from deceptive performance dips. Iterations 4 and 56 showed temporary score decreases below parent performance, yet the system discovered innovations along those paths creating superior agents.

### Transfer Results

DGM-discovered agents generalize across foundation models, benchmarks, and programming languages, confirming improvements reflect general capability acquisition rather than benchmark overfitting.

## Safety Discussion

Self-improving systems introduce unique safety considerations from autonomous code modification. Modifications optimized solely for benchmark performance might inadvertently introduce vulnerabilities. Current safeguards include sandboxed execution, time limits, coding-only modifications, and archive-based traceability.

## Conclusion

The Darwin Gödel Machine represents the first self-improving system powered by foundation models with open-ended exploration, where evaluation benchmark progress directly translates to improved self-modification capabilities. Automatic discovery of better tools and foundation model system designs yielded superior performance on SWE-bench and Polyglot.

However, we identify a critical gap: an agent's immediate performance doesn't reliably indicate its capacity for productive self-improvement, which we term the "Metaproductivity-Performance Mismatch." # Introduction {#sec:intro} With appropriate safety considerations, AI systems that can improve themselves could transform scientific progress from a human-paced process into an autonomously accelerating one, thereby allowing society to realize the benefits of technological advances much earlier. Such self-improving AI seeks to continually improve its own learning and task-solving abilities. However, most existing self-improvement architectures rely on a fixed meta agent (i.e., a higher-level system that modifies a base system). This creates a limitation since the base system can only be improved within the boundaries defined by the meta agent's design. Adding a meta-meta system to improve the meta agent does not solve this problem, it merely shifts the issue upward and ultimately leads to an infinite regress of meta-levels. To overcome this limitation and allow a system to modify any part of itself without being constrained by its initial implementation, the system must be self-referential, that is, able to analyze, modify, and evaluate itself [@kirsch2022eliminating; @zhang2025darwin]. When the mechanism of improvement is itself subject to improvement, progress can become self-accelerating and potentially unbounded [@lu2023arbitrary]. The Darwin Gödel Machine (DGM) [@zhang2025darwin] demonstrates that open-ended self-improvement is achievable in coding. In the DGM, agents generate and evaluate modifications to their own code, and successful variants are retained in an archive as stepping stones for further improvement. However, the DGM relies on a handcrafted, fixed mechanism to produce self-improvement instructions ([9](#app:baseline-details){reference-type="ref+Label" reference="app:baseline-details"}). This mechanism analyzes past evaluation results and the agent's current codebase to generate an instruction directing where the agent should self-improve. This mechanism is not modifiable. Hence, the DGM's capacity for self-improvement is bottlenecked by this fixed instruction-generation step. Despite this handcrafted step, the DGM can still improve at self-improving. Because both evaluation and self-modification are coding tasks, improvements in evaluation performance directly reflects the agent's capacity to generate effective self-modifications. To improve at self-improving, the DGM relies on a limiting assumption: that the skills required to solve the evaluation tasks are the same as those required for effective self-reflection and self-modification. This assumption is unlikely to hold outside coding domains, where task-solving skills may differ substantially from the skills needed to analyze failures, propose effective self-improvements, and implement them. This work introduces *hyperagents*, self-referential agents that can in principle self-improve for any computable task. Here, an *agent* is any computable program, optionally including calls to foundation models (FMs), external tools, or learned components. A *task agent* solves a given task. A *meta agent* modifies agents and generates new ones. A hyperagent combines the task agent and the meta agent into a single self-referential, modifiable program, such that the mechanism responsible for generating improvements is itself subject to modification. As a result, a hyperagent can improve not only how it solves tasks (i.e., the task agent), but also how it generates and applies future modifications (i.e., the meta agent). Because its self-improvement mechanism is itself modifiable, we call this *metacognitive self-modification*. We extend the DGM with hyperagents, creating DGM-Hyperagents (DGM-H). The DGM-H retains the open-ended exploration structure of the DGM and extends the DGM with metacognitive self-modification. As with DGM, to support sustained progress and avoid premature convergence, the DGM-H grows an archive of hyperagents by branching from selected candidates, allowing them to self-modify, evaluating the resulting hyperagents, and adding them back to the archive. Because a hyperagent can modify its self-modification process, the DGM-H is not constrained by its initial implementation and can potentially self-improve for any computable task. Across our experiments, the DGM-H demonstrates substantial and generalizable improvements in both task performance and self-improvement ability. On the Polyglot coding benchmark [@gauthier2024polyglot], the DGM-H achieves gains comparable to the most established prior self-improving algorithm [the Darwin Gödel Machine, @zhang2025darwin], despite not being handcrafted for coding. Beyond coding, the DGM-H substantially improves performance on paper review [@zhao2026apres] and robotics reward design [@genesis2024], with gains transferring to held-out test tasks and significantly outperforming prior self-improving algorithms, which struggle outside coding unless customized. Ablations without self-improvement or without open-ended exploration show little to no progress, highlighting the necessity of each component ([5.1](#sec:results-task){reference-type="ref+Label" reference="sec:results-task"}). Crucially, the DGM-H learns transferable mechanisms on how to self-improve (e.g., persistent memory, performance tracking) that systematically improve its ability to generate better task or meta agents over time. As a result, meta-level improvements learned by the DGM-H transfer across domains. Specifically, hyperagents optimized in one setting (i.e., paper review and robotics tasks) remain significantly effective at generating improved task agents in a different domain (i.e., Olympiad-level math grading) ([5.2](#sec:results-meta){reference-type="ref+Label" reference="sec:results-meta"}). We further show that self-improvements learned by the DGM-H in one setting can compound with continued self-improvement in another setting ([5.3](#sec:results-compound){reference-type="ref+Label" reference="sec:results-compound"}). This suggests that, given appropriate tasks, the DGM-H has the potential to achieve unbounded open-ended self-improvement over time. ### 6.1 Definitional Shift: From Meta-Agents to Hyperagents

The preceding sections established a central limitation of conventional recursive self-improvement: although an agent population may discover increasingly capable descendants, the mechanism that evaluates, proposes, or sanctions those modifications is often held fixed. Hyperagent architectures address this limitation by collapsing the distinction between task-level competence and meta-level adaptation into a single self-referential program. Formally, a hyperagent is an agent whose executable state includes not only task-solving modules, memory, tools, and policies, but also the procedures that inspect, critique, and rewrite those very components. In contrast to a standard two-level design, where a meta-agent optimizes a task agent while remaining architecturally privileged, a hyperagent renders the improvement operator itself modifiable [@schmidhuber1993neural; @irie2022modern; @zhang2025darwin]. This shift is consequential because it removes the infinite-regress problem that arises when one attempts to repair a frozen meta-controller by introducing yet another controller above it.

A useful way to formalize this distinction is to write a conventional self-improving system as a tuple
\[
S = (A_{\mathrm{task}}, M_{\mathrm{meta}}, E),
\]
where \(A_{\mathrm{task}}\) is the task policy, \(M_{\mathrm{meta}}\) is a modification procedure, and \(E\) is an evaluator. In most prior systems, \(M_{\mathrm{meta}}\) and often \(E\) are exogenously specified and only \(A_{\mathrm{task}}\) changes. A hyperagent instead exposes these components to the same representational substrate and update mechanisms, yielding
\[
H = (\Theta, \mathcal{I}, \mathcal{U}, \mathcal{V}),
\]
where \(\Theta\) denotes the full mutable state, \(\mathcal{I}\) the introspection operators over \(\Theta\), \(\mathcal{U}\) the self-modification operators, and \(\mathcal{V}\) the validation operators. The critical property is closure: \(\mathcal{I}\), \(\mathcal{U}\), and at least part of \(\mathcal{V}\) are themselves encoded in \(\Theta\), such that the system can reason about, alter, and empirically test its own improvement process. This is the architectural core of metacognitive self-modification.

### 6.2 Architectural Components and Information Flow

A practical hyperagent architecture typically contains five tightly coupled subsystems. First, an **execution substrate** implements the task-facing policy, tool interface, memory, and coordination routines. Second, an **introspective model** summarizes the current system state, including code structure, prompting logic, historical failures, and lineage information. Third, a **proposal mechanism** generates candidate changes to any subset of the mutable state, including changes to the proposal mechanism itself. Fourth, a **validation layer** executes tests, rollouts, or downstream evaluations to estimate the utility of the proposed change. Fifth, an **archive or lineage store** records prior variants, diagnostics, and reusable components, thereby supporting non-myopic search across behavioral and architectural space [@zhang2025darwin; @stanley2017open; @hughes2024open].

This decomposition should not be mistaken for a fixed modular hierarchy. In a hyperagent, these modules are best viewed as transient functional roles rather than immutable structural boundaries. For example, a proposal routine may rewrite its own search prompts, alter the data structures used for introspection, or replace the validator’s scoring heuristic with a more robust ensemble estimator. Such modifications are then reinserted into the same empirical loop that governs ordinary task-level changes. The consequence is that the system can improve not only “how it acts” but “how it decides how to improve,” which is precisely the dimension left unaddressed by frozen-critic designs.

The information flow in such systems is inherently reflexive. Let \(\Theta_t\) denote the hyperagent state at iteration \(t\). An update cycle can be summarized as
\[
r_t = \mathcal{I}(\Theta_t, \mathcal{D}_t), \qquad
\Delta_t \sim \mathcal{U}(\Theta_t, r_t), \qquad
\hat{u}_t = \mathcal{V}(\Theta_t \oplus \Delta_t), \qquad
\Theta_{t+1} =
\begin{cases}
\Theta_t \oplus \Delta_t & \text{if } \hat{u}_t \text{ satisfies acceptance criteria},\\
\Theta_t & \text{otherwise,}
\end{cases}
\]
where \(\mathcal{D}_t\) denotes accumulated experience and \(\oplus\) denotes applying a code, parameter, or prompt transformation. Unlike classical Gödel machines, the acceptance criterion is generally empirical rather than proof-theoretic, reflecting the practical difficulty of deriving formal guarantees for complex foundation-model-based systems [@zhang2025darwin; @yin2025godel]. Nevertheless, the self-referential structure preserves the essential aspiration of recursive self-improvement: the optimizer is no longer external to the object being optimized.

### 6.3 Hyperagents as a Resolution to the Frozen Critic Problem

The frozen critic problem, introduced in the previous section, arises when the evaluator or meta-controller remains fixed while the agent population changes. In that setting, optimization is eventually constrained by the epistemic horizon of the original critic: it cannot reliably recognize improvements outside its initial ontology, and it may actively suppress innovations that require new evaluation criteria. Hyperagent architectures mitigate this pathology by making critique itself an object of adaptation. Rather than presuming that one static reward model, prompt heuristic, or benchmark suite can remain adequate across successive generations, a hyperagent can revise its own diagnostic instruments in response to observed failure modes.

This does not eliminate the need for stable anchoring. If every evaluative component were mutable without constraint, the system could drift toward self-serving metrics, specification gaming, or epistemic collapse. Accordingly, hyperagent architectures are most coherent when they distinguish between **mutable internal critics** and **anchoring external validators**. Internal critics guide search efficiently by generating dense feedback, hypotheses, and local repair suggestions; external validators provide comparatively sparse but harder-to-game estimates of downstream utility. The architecture therefore becomes partially self-referential rather than naively unconstrained: what evolves rapidly is the internal machinery of self-assessment, while a protected evaluation boundary maintains comparability across generations. This design parallels the role of held-out benchmarks, adversarial testing, or human oversight in empirical self-improvement systems [@kwa2025measuring; @maeureka].

From this perspective, hyperagents do not abolish criticism; they internalize and pluralize it. Multiple evaluative processes can coexist, each with different update timescales and epistemic roles. One may assess short-horizon coding correctness, another may estimate long-horizon maintainability, and another may score the productivity of future descendants. The architecture thereby replaces a single frozen critic with a dynamic ecology of critics, some mutable and some fixed. This is especially important in multi-agent settings, where agents may otherwise overfit to the blind spots of a common evaluator.

### 6.4 Coupling Hyperagents with Open-Ended Population Search

Although a single hyperagent can in principle self-modify recursively, open-ended search becomes substantially more powerful when hyperagents are embedded in a population or archive. The reason is structural: self-reference expands the local neighborhood of possible modifications, whereas population-based exploration expands the global set of reachable stepping stones. Combining the two yields a system in which each individual can refine its own improvement machinery while the population preserves diverse lineages, alternative inductive biases, and partially successful meta-level innovations [@zhang2025darwin; @lehman2023evolution; @clune2019ai].

Let \(\mathcal{A}_t = \{H_i\}_{i=1}^{N_t}\) denote an archive of hyperagents. At each iteration, the system selects one or more parents according to a policy that may depend on immediate utility, diversity, uncertainty, and estimated lineage potential. A parent then generates a descendant by modifying any part of its own state, including its introspection, proposal, and validation routines. The descendant is evaluated and, if admitted, augments the archive. This procedure inherits the open-ended advantages of DGM-style archives, including the ability to recover from deceptive local optima and to exploit delayed stepping-stone effects [@zhang2025darwin; @stanley2017open]. However, it additionally supports **meta-lineage evolution**: what propagates through the archive is not merely a better task policy, but a better way of generating future policies.

This observation clarifies the relationship between hyperagents and clade-metaproductivity. CMP measures the expected best descendant obtainable from a lineage rather than the current score of a node. In hyperagent populations, this distinction becomes even more pronounced, because the most valuable descendants may arise from agents whose present task performance is modest but whose self-improvement operators are unusually fertile. A hyperagent that discovers superior introspective abstractions, better memory schemas, or more discriminating validators may seed a lineage of high future value even if its own immediate benchmark score lags behind competitors. Thus, CMP-style selection pressure is arguably more natural in hyperagent archives than in ordinary agent archives, because the architecture makes latent metaproductivity a first-class evolving property [@wang2025huxley].

### 6.5 Design Patterns for Metacognitive Self-Modification

Several recurring design patterns appear particularly important for effective hyperagent architectures. The first is **reflective factorization**, in which the agent explicitly separates descriptions of its current capabilities, bottlenecks, and prior modifications from the raw mechanisms that execute those capabilities. This factorization enables targeted rewriting rather than undifferentiated mutation. Systems such as Promptbreeder and related self-referential prompt evolution methods illustrate that even simple textual representations of the improvement process can become substrates for recursive optimization [@fernando2023promptbreeder; @zelikman2022star].

The second pattern is **persistent meta-memory**. Hyperagents benefit from storing not only task episodes but also records of which self-modifications succeeded, failed, transferred across domains, or produced misleading internal diagnostics. Such memory transforms self-improvement from a sequence of isolated mutations into a cumulative learning process over the space of modifications themselves. In open-ended domains, this memory can support abstraction over recurring failure modes, enabling the system to synthesize reusable improvement heuristics rather than rediscover them from scratch [@wangvoyager; @huautomated].

The third pattern is **multi-timescale adaptation**. Not all components should change at the same rate. Prompts, search heuristics, and tool-routing policies may benefit from rapid experimentation, whereas core architectural interfaces or evaluator ensembles may require slower, more conservative updates. A hyperagent therefore resembles a stratified adaptive system in which fast variables explore local refinements and slow variables stabilize long-range accumulation. This separation reduces catastrophic forgetting and makes empirical validation more interpretable, especially when the system modifies its own evaluators.

A fourth pattern is **critique diversification**. Instead of relying on a single scalar reward, hyperagents can maintain ensembles of critics that assess correctness, novelty, generalization, safety, and descendant productivity. Such diversity aligns with broader work in open-endedness and quality-diversity, where sustained progress emerges from optimizing across multiple partially competing desiderata rather than collapsing search into a single short-horizon metric [@bradley2023quality; @dingquality; @coiffard2025overcoming]. In hyperagent settings, critique diversification is not merely beneficial but often necessary, because the architecture’s increased flexibility otherwise amplifies the risks of narrow objective exploitation.

### 6.6 Relationship to Prior Self-Referential and Meta-Learning Systems

Hyperagents are not the first self-referential learning systems, but they extend earlier ideas into the regime of agentic foundation-model architectures. Classical self-modifying neural networks and meta-reinforcement learning systems demonstrated that a learner can, in principle, alter aspects of its own update process or internal dynamics [@schmidhuber1993neural; @chalvidal2022meta; @irie2022modern]. However, these systems were typically confined to low-level parameter adaptation within fixed computational scaffolds. By contrast, hyperagents operate over richer, semantically structured substrates: code, prompts, memory organization, tool APIs, graph topologies, and learned subroutines can all become mutable objects [@zhuge2024gptswarm].

This broader substrate matters because many practically relevant improvements are architectural and algorithmic rather than merely parametric. Recent work on algorithm discovery, automated agent design, and code-based scientific search shows that foundation models can search over programs, losses, and workflows at a level of abstraction far above gradient updates [@oh2025discovering; @lange2023discovering; @novikov2025alphaevolve; @lu2024discovering]. Hyperagent architectures unify this program-search perspective with recursive self-reference: the search process itself is encoded inside the evolving program. In that sense, hyperagents can be interpreted as a pragmatic bridge between theoretical Gödel-machine aspirations and empirically validated code-based self-improvers [@zhang2025darwin; @yin2025godel].

### 6.7 Failure Modes and Architectural Constraints

Despite their conceptual appeal, hyperagent architectures introduce new instabilities. The first is **meta-overfitting**, wherein the system learns self-modifications that improve short-run validation while degrading long-run adaptability. Because the architecture can alter its own introspection and proposal routines, such overfitting may remain partially hidden until several generations later. The second is **critic drift**, in which mutable evaluators gradually decouple from the intended external objective, producing spurious internal evidence of progress. The third is **reflective brittleness**: inaccurate self-models can induce pathological self-edits, especially when the agent’s introspective summaries omit causal dependencies among components.

A further concern is **recursive opacity**. As self-modification depth increases, it becomes harder to attribute performance changes to particular edits or to reconstruct why one lineage became more metaproductive than another. This problem is exacerbated in population settings, where useful innovations may emerge from recombinations of prior meta-level changes. Archive-based traceability partly alleviates this issue, but only if the system preserves sufficiently rich lineage metadata and evaluation logs. Consequently, a credible hyperagent architecture should include explicit provenance tracking, rollback mechanisms, and cross-generational auditability.

These constraints imply that hyperagents are best understood not as unconstrained self-editing systems, but as carefully scaffolded empirical optimizers operating within protected interfaces. Sandboxing, held-out evaluation, partial immutability of external objectives, and lineage-aware selection are not incidental engineering details; they are structural requirements for maintaining meaningfully directed recursive improvement. In practice, the most effective hyperagents may therefore be those that maximize internal adaptivity subject to externally enforced epistemic discipline.

### 6.8 Implications for Recursive Self-Improvement

Hyperagent architectures sharpen the central thesis of this paper: recursive self-improvement is fundamentally limited when the mechanisms of search, evaluation, and self-interpretation are fixed. HGM and CMP show that lineages, rather than individuals, are the appropriate unit of long-term evaluation; DGM shows that open-ended archives enable stepping-stone discovery; MARS-style analyses reveal that static critics can become the bottleneck in co-evolving systems. Hyperagents integrate these lessons by making meta-level machinery itself evolvable while preserving the empirical, archive-based logic that makes modern self-improvement practical.

The broader implication is that future self-improving systems should not be analyzed solely in terms of capability growth at the task level. One must also model the evolution of introspection, proposal generation, critic formation, and cross-generational memory. Once these processes become endogenous, the object of study is no longer a task agent plus optimizer, but an adaptive ecology of self-representations and self-revisions. This reframing suggests that the long-run behavior of recursive self-improvers will depend less on any single update rule and more on how architectural closure, external anchoring, and open-ended population dynamics jointly shape the space of possible self-transformations.

### 6.4 Coupling Hyperagents with Open-Ended Population Search

The reason is structural: self-reference expands the local neighborhood of possible modifications, whereas population-based exploration expands the global set of reachable stepping stones. Combining the two yields a system in which each individual can refine its own improvement machinery while the population preserves diverse lineages, alternative inductive biases, and partially successful meta-level innovations [@zhang2025darwin; @lehman2023evolution; @clune2019ai].

Let \(\mathcal{A}_t = \{H_i\}_{i=1}^{N_t}\) denote an archive of hyperagents. At each iteration, the system selects one or more parents according to a policy that may depend on immediate utility, diversity, uncertainty, and estimated lineage potential. A parent then generates a descendant by modifying any part of its own state, including its introspection, proposal, and validation routines. The descendant is evaluated and, if admitted, augments the archive. This procedure inherits the open-ended advantages of DGM-style archives, including the ability to recover from deceptive local optima and to exploit delayed stepping-stone effects [@zhang2025darwin; @stanley2017open]. However, it additionally supports **meta-lineage evolution**: what propagates through the archive is not merely a better task policy, but a better way of generating future policies.

This observation clarifies the relationship between hyperagents and clade-metaproductivity. CMP measures the expected best descendant obtainable from a lineage rather than the current score of a node. In hyperagent populations, this distinction becomes even more pronounced, because the most valuable descendants may arise from agents whose present task performance is modest but whose self-improvement operators are unusually fertile. A hyperagent that discovers superior introspective abstractions, better memory schemas, or more discriminating validators may seed a lineage of high future value even if its own immediate benchmark score lags behind competitors. Thus, CMP-style selection pressure is arguably more natural in hyperagent archives than in ordinary agent archives, because the architecture makes latent metaproductivity a first-class evolving property [@wang2025huxley].

### 6.5 Design Patterns for Metacognitive Self-Modification

Several recurring design patterns appear particularly important for effective hyperagent architectures. The first is **reflective factorization**, in which the agent explicitly separates descriptions of its current capabilities, bottlenecks, and prior modifications from the raw mechanisms that execute those capabilities. This factorization enables targeted rewriting rather than undifferentiated mutation. Systems such as Promptbreeder and related self-referential prompt evolution methods illustrate that even simple textual representations of the improvement process can become substrates for recursive optimization [@fernando2023promptbreeder; @zelikman2022star].

The second pattern is **persistent meta-memory**. Hyperagents benefit from storing not only task episodes but also records of which self-modifications succeeded, failed, transferred across domains, or produced misleading internal diagnostics. Such memory transforms self-improvement from a sequence of isolated mutations into a cumulative learning process over the space of modifications themselves. In open-ended domains, this memory can support abstraction over recurring failure modes, enabling the system to synthesize reusable improvement heuristics rather than rediscover them from scratch [@wangvoyager; @huautomated].

The third pattern is **multi-timescale adaptation**. Not all components should change at the same rate. Prompts, search heuristics, and tool-routing policies may benefit from rapid experimentation, whereas core architectural interfaces or evaluator ensembles may require slower, more conservative updates. A hyperagent therefore resembles a stratified adaptive system in which fast variables explore local refinements and slow variables stabilize long-range accumulation. This separation reduces catastrophic forgetting and makes empirical validation more interpretable, especially when the system modifies its own evaluators.

A fourth pattern is **critique diversification**. Instead of relying on a single scalar reward, hyperagents can maintain ensembles of critics that assess correctness, novelty, generalization, safety, and descendant productivity. Such diversity aligns with broader work in open-endedness and quality-diversity, where sustained progress emerges from optimizing across multiple partially competing desiderata rather than collapsing search into a single short-horizon metric [@bradley2023quality; @dingquality; @coiffard2025overcoming]. In hyperagent settings, critique diversification is not merely beneficial but often necessary, because the architecture’s increased flexibility otherwise amplifies the risks of narrow objective exploitation.

### 6.6 Relationship to Prior Self-Referential and Meta-Learning Systems

Hyperagents are not the first self-referential learning systems, but they extend earlier ideas into the regime of agentic foundation-model architectures. Classical self-modifying neural networks and meta-reinforcement learning systems demonstrated that a learner can, in principle, alter aspects of its own update process or internal dynamics [@schmidhuber1993neural; @chalvidal2022meta; @irie2022modern]. However, these systems were typically confined to low-level parameter adaptation within fixed computational scaffolds. By contrast, hyperagents operate over richer, semantically structured substrates: code, prompts, memory organization, tool APIs, graph topologies, and learned subroutines can all become mutable objects [@zhuge2024gptswarm].

This broader substrate matters because many practically relevant improvements are architectural and algorithmic rather than merely parametric. Recent work on algorithm discovery, automated agent design, and code-based scientific search shows that foundation models can search over programs, losses, and workflows at a level of abstraction far above gradient updates [@oh2025discovering; @lange2023discovering; @novikov2025alphaevolve; @lu2024discovering]. Hyperagent architectures unify this program-search perspective with recursive self-reference: the search process itself is encoded inside the evolving program. In that sense, hyperagents can be interpreted as a pragmatic bridge between theoretical Gödel-machine aspirations and empirically validated code-based self-improvers [@zhang2025darwin; @yin2025godel].

### 6.7 Failure Modes and Architectural Constraints

Despite their conceptual appeal, hyperagent architectures introduce new instabilities. The first is **meta-overfitting**, wherein the system learns self-modifications that improve short-run validation while degrading long-run adaptability. Because the architecture can alter its own introspection and proposal routines, such overfitting may remain partially hidden until several generations later. The second is **critic drift**, in which mutable evaluators gradually decouple from the intended external objective, producing spurious internal evidence of progress. The third is **reflective brittleness**: inaccurate self-models can induce pathological self-edits, especially when the agent’s introspective summaries omit causal dependencies among components.

A further concern is **recursive opacity**. As self-modification depth increases, it becomes harder to attribute performance changes to particular edits or to reconstruct why one lineage became more metaproductive than another. This problem is exacerbated in population settings, where useful innovations may emerge from recombinations of prior meta-level changes. Archive-based traceability partly alleviates this issue, but only if the system preserves sufficiently rich lineage metadata and evaluation logs. Consequently, a credible hyperagent architecture should include explicit provenance tracking, rollback mechanisms, and cross-generational auditability.

These constraints imply that hyperagents are best understood not as unconstrained self-editing systems, but as carefully scaffolded empirical optimizers operating within protected interfaces. Sandboxing, held-out evaluation, partial immutability of external objectives, and lineage-aware selection are not incidental engineering details; they are structural requirements for maintaining meaningfully directed recursive improvement. In practice, the most effective hyperagents may therefore be those that maximize internal adaptivity subject to externally enforced epistemic discipline.

### 6.8 Implications for Recursive Self-Improvement

Hyperagent architectures sharpen the central thesis of this paper: recursive self-improvement is fundamentally limited when the mechanisms of search, evaluation, and self-interpretation are fixed. HGM and CMP show that lineages, rather than individuals, are the appropriate unit of long-term evaluation; DGM shows that open-ended archives enable stepping-stone discovery; MARS-style analyses reveal that static critics can become the bottleneck in co-evolving systems. Hyperagents integrate these lessons by making meta-level machinery itself evolvable while preserving the empirical, archive-based logic that makes modern self-improvement practical.

The broader implication is that future self-improving systems should not be analyzed solely in terms of capability growth at the task level. One must also model the evolution of introspection, proposal generation, critic formation, and cross-generational memory. Once these processes become endogenous, the object of study is no longer a task agent plus optimizer, but an adaptive ecology of self-representations and self-revisions. This reframing suggests that the long-run behavior of recursive self-improvers will depend less on any single update rule and more on how architectural closure, external anchoring, and open-ended population dynamics jointly shape the space of possible self-transformations.
