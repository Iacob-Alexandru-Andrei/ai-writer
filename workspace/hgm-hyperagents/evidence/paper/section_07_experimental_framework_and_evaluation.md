# Darwin Gödel Machine: Open-Ended Evolution of Self-Improving Agents

Jenny Zhang, Shengran Hu, Cong Lu, Robert Lange, Jeff Clune

University of British Columbia, Vector Institute, Sakana AI

## Abstract

Most contemporary AI systems operate within fixed, human-designed architectures and cannot autonomously improve themselves. We introduce the Darwin Gödel Machine (DGM), a self-referential system that iteratively modifies its own code to enhance coding capabilities. Rather than requiring formal proofs of beneficial changes (as theoretical Gödel machines demand), the DGM empirically validates modifications using coding benchmarks. The system maintains an archive of generated agents and samples from it for self-modification, inspired by Darwinian evolution and open-endedness principles. Results demonstrate performance improvements from 20.0% to 50.0% on SWE-bench and 14.2% to 30.7% on Polyglot, significantly outperforming baselines lacking self-improvement or open-ended exploration.

## Introduction

Scientific progress operates cumulatively, with innovations building on prior discoveries. Current AI systems, despite their sophistication, remain constrained by fixed architectures designed by humans. Unlike the scientific method, which enables continuous self-directed advancement, today's AI development still depends heavily on human intervention. This paper investigates automating the search for progressively better AI systems.

Schmidhuber's theoretical Gödel machines represent one approach, relying on formal proofs ensuring modifications improve performance. However, proving that changes benefit complex systems like large language models is practically impossible. The actual impact of modifications depends heavily on context---a testing tool optimized for one environment might confuse an agent in another.

The Darwin Gödel Machine relaxes this requirement, instead using empirical validation through benchmarking. This mirrors biological evolution, where mutations aren't verified in advance but produced, tested, and selected through natural selection. The DGM combines this with inspiration from Darwinian principles and open-endedness research, maintaining a library of discovered agents as stepping stones for future generations.

## Darwin Gödel Machine

A Gödel Machine theoretically searches for provably beneficial self-improvements. The DGM relaxes this impractical requirement, instead requiring empirical evidence from experiments. This empiricism addresses the impossibility of proving changes improve complex systems.

Since empirical evidence can trap systems in local optima within vast algorithmic search spaces, the DGM maintains an archive of discovered solutions, enabling open-ended exploration rather than single-solution evolution. This mirrors Darwinian evolution, where innovations emerge by selecting entities from previous discoveries, modifying them, and retaining interesting variations.

### Self-Referential Self-Improvement of Coding Agents

The DGM initializes with one coding agent and evaluates progress on coding benchmarks. A coding agent represents a system implemented as a code repository, powered by frozen pretrained foundation models, capable of reading, writing, and executing code.

Self-improvement is defined as modifying the agent's own codebase components---tools, workflows, prompts. Python's Turing-completeness enables building any computable machine. The key motivation: empirical improvements must reflect both downstream task-solving ability and self-modification capacity. By configuring the DGM as a coding agent and testing coding capabilities, observed improvements demonstrate enhanced problem-solving and future self-improvement potential.

### Population-Based Open-Ended Exploration

Starting from a single agent, the DGM accumulates all generated variants in an archive. Each iteration selects parent agents for self-modification, roughly proportional to performance and inversely proportional to the number of their children. This balances exploitation of strong performers with exploration of promising, undersampled lineages.

All agents retain non-zero selection probability, ensuring any improvement pathway remains feasible. Each selected parent analyzes benchmark evaluation logs, proposes the next feature, and executes the modification. Newly generated agents undergo quantitative benchmark evaluation. Only agents compiling successfully and retaining codebase-editing ability join the archive; others are discarded.

Crucially, archived solutions can later serve as stepping stones for improvements discovered much later---substantially differentiating this from hill-climbing approaches.

## Experiments

### Setup

The DGM initializes with a lightweight coding agent built around a frozen foundation model with tool-use capabilities. The base agent accesses two tools: a Bash tool for command execution and an edit tool for viewing and modifying files. The intentionally lightweight design provides a simple starting point demonstrating clear improvement.

The DGM ran for 80 iterations, generating one new agent per iteration, with parallelization. During self-modifications, agents used Claude 3.5 Sonnet.

### Results

After 80 iterations, coding agent performance increased from 20.0% to 50.0% on SWE-bench and from 14.0% to 38.0% on Polyglot. DGM-discovered agents achieve performance comparable to or exceeding handcrafted solutions.

The DGM automatically enhanced both tools and foundation model workflows. Tool improvements included granular file viewing and precise editing. Workflow improvements encompassed multiple solution attempts and using another FM to evaluate and select optimal solutions.

Open-ended exploration allows escape from deceptive performance dips. Iterations 4 and 56 showed temporary score decreases below parent performance, yet the system discovered innovations along those paths creating superior agents.

### Transfer Results

DGM-discovered agents generalize across foundation models, benchmarks, and programming languages, confirming improvements reflect general capability acquisition rather than benchmark overfitting.

## Safety Discussion

Self-improving systems introduce unique safety considerations from autonomous code modification. Modifications optimized solely for benchmark performance might inadvertently introduce vulnerabilities. Current safeguards include sandboxed execution, time limits, coding-only modifications, and archive-based traceability.

## Conclusion

The Darwin Gödel Machine represents the first self-improving system powered by foundation models with open-ended exploration, where evaluation benchmark progress directly translates to improved self-modification capabilities. Automatic discovery of better tools and foundation model system designs yielded superior performance on SWE-bench and Polyglot.

### Example 3: hyperagents.tex

# Introduction {#sec:intro} With appropriate safety considerations, AI systems that can improve themselves could transform scientific progress from a human-paced process into an autonomously accelerating one, thereby allowing society to realize the benefits of technological advances much earlier. Such self-improving AI seeks to continually improve its own learning and task-solving abilities. However, most existing self-improvement architectures rely on a fixed meta agent (i.e., a higher-level system that modifies a base system). This creates a limitation since the base system can only be improved within the boundaries defined by the meta agent's design. Adding a meta-meta system to improve the meta agent does not solve this problem, it merely shifts the issue upward and ultimately leads to an infinite regress of meta-levels. To overcome this limitation and allow a system to modify any part of itself without being constrained by its initial implementation, the system must be self-referential, that is, able to analyze, modify, and evaluate itself [@kirsch2022eliminating; @zhang2025darwin]. When the mechanism of improvement is itself subject to improvement, progress can become self-accelerating and potentially unbounded [@lu2023arbitrary]. The Darwin Gödel Machine (DGM) [@zhang2025darwin] demonstrates that open-ended self-improvement is achievable in coding. In the DGM, agents generate and evaluate modifications to their own code, and successful variants are retained in an archive as stepping stones for further improvement. However, the DGM relies on a handcrafted, fixed mechanism to produce self-improvement instructions ([9](#app:baseline-details){reference-type="ref+Label" reference="app:baseline-details"}). This mechanism analyzes past evaluation results and the agent's current codebase to generate an instruction directing where the agent should self-improve. This mechanism is not modifiable. Hence, the DGM's capacity for self-improvement is bottlenecked by this fixed instruction-generation step. Despite this handcrafted step, the DGM can still improve at self-improving. Because both evaluation and self-modification are coding tasks, improvements in evaluation performance directly reflects the agent's capacity to generate effective self-modifications. To improve at self-improving, the DGM relies on a limiting assumption: that the skills required to solve the evaluation tasks are the same as those required for effective self-reflection and self-modification. This assumption is unlikely to hold outside coding domains, where task-solving skills may differ substantially from the skills needed to analyze failures, propose effective self-improvements, and implement them. This work introduces *hyperagents*, self-referential agents that can in principle self-improve for any computable task. Here, an *agent* is any computable program, optionally including calls to foundation models (FMs), external tools, or learned components. A *task agent* solves a given task. A *meta agent* modifies agents and generates new ones. A hyperagent combines the task agent and the meta agent into a single self-referential, modifiable program, such that the mechanism responsible for generating improvements is itself subject to modification. As a result, a hyperagent can improve not only how it solves tasks (i.e., the task agent), but also how it generates and applies future modifications (i.e., the meta agent). Because its self-improvement mechanism is itself modifiable, we call this *metacognitive self-modification*. We extend the DGM with hyperagents, creating DGM-Hyperagents (DGM-H). The DGM-H retains the open-ended exploration structure of the DGM and extends the DGM with metacognitive self-modification. As with DGM, to support sustained progress and avoid premature convergence, the DGM-H grows an archive of hyperagents by branching from selected candidates, allowing them to self-modify, evaluating the resulting hyperagents, and adding them back to the archive. Because a hyperagent can modify its self-modification process, the DGM-H is not constrained by its initial implementation and can potentially self-improve for any computable task. Across our experiments, the DGM-H demonstrates substantial and generalizable improvements in both task performance and self-improvement ability. On the Polyglot coding benchmark [@gauthier2024polyglot], the DGM-H achieves gains comparable to the most established prior self-improving algorithm [the Darwin Gödel Machine, @zhang2025darwin], despite not being handcrafted for coding. Beyond coding, the DGM-H substantially improves performance on paper review [@zhao2026apres] and robotics reward design [@genesis2024], with gains transferring to held-out test tasks and significantly outperforming prior self-improving algorithms, which struggle outside coding unless customized. Ablations without self-improvement or without open-ended exploration show little to no progress, highlighting the necessity of each component ([5.1](#sec:results-task){reference-type="ref+Label" reference="sec:results-task"}). Crucially, the DGM-H learns transferable mechanisms on how to self-improve (e.g., persistent memory, performance tracking) that systematically improve its ability to generate better task or meta agents over time. ### 8.1 Evaluation Objectives

The central empirical question is not merely whether a self-improving system can obtain higher task scores over time, but whether it can improve *how it improves* under realistic budget, stochasticity, and distribution-shift constraints. Accordingly, we evaluate four properties in parallel: immediate task competence, metaproductivity of descendants, robustness to frozen or partially frozen critics, and transfer of self-improvement mechanisms across domains. This design follows the preceding theoretical claim that single-agent performance is an incomplete proxy for long-run improvement potential, and that archive-based or hyperagent-based systems should be assessed at the lineage level rather than only at the level of individual checkpoints [@zhang2025darwin; @wang2025huxley; @hughes2024open]. The resulting framework is intentionally comparative: each method is exposed to the same computational budget, model family, and evaluation schedule, with only the self-improvement algorithm varied.

Our hypotheses are fourfold. First, CMP-guided selection should outperform greedy single-agent hill-climbing when the search landscape is deceptive, because high-performing individuals need not be high-yield ancestors [@wang2025huxley; @coiffard2025overcoming]. Second, DGM-style archive exploration should dominate purely exploitative baselines in open-ended search spaces by preserving stepping-stone variants that appear locally suboptimal yet seed later gains [@zhang2025darwin; @stanley2017open]. Third, MARS-style co-evolution should improve adaptation only when critic evolution is sufficiently coupled to actor evolution; otherwise, the frozen critic problem should induce premature stagnation. Fourth, hyperagent architectures should exhibit the strongest cross-domain transfer because they can modify not only task policies but also the mechanisms by which future modifications are proposed, filtered, and retained [@schmidhuber1993neural; @irie2022modern; @yin2025godel].

### 8.2 Task Suite and Environments

We use a heterogeneous evaluation suite designed to separate narrow task overfitting from genuine self-improvement capacity. The primary coding domain uses repository-level software engineering tasks, following recent work on self-improving coding agents and automated design of agentic systems [@zhang2025darwin; @huautomated]. We recommend SWE-bench-style issue resolution and Polyglot-style multilingual programming tasks as the main downstream benchmarks because they are executable, naturally support repeated evaluation, and align with the self-modification interface used by current coding agents. Coding remains the cleanest initial domain because the agent, the artifact being modified, and the evaluation harness all share a common programmatic substrate.

To test whether the learned improvement mechanisms are domain-general rather than coding-specific, we include two non-coding settings. The first is paper-review or long-horizon evaluation assistance, where the agent must synthesize evidence, identify methodological flaws, and produce structured judgments under rubric-based supervision; this stresses deliberative evaluation and memory rather than direct code manipulation. The second is reward-design or algorithm-design for embodied control, where the agent proposes reward functions or training scaffolds that are subsequently evaluated by an external simulator or learning system, following the broader paradigm of LLM-mediated design search [@maeureka; @oh2025discovering; @lange2023discovering]. Together, these domains induce variation in feedback latency, observability, and edit granularity, which is necessary for diagnosing when apparent recursive self-improvement is merely benchmark specialization.

For each domain, we partition tasks into three disjoint subsets: improvement tasks, validation tasks, and transfer tasks. Improvement tasks are visible to the self-improving system and generate the rewards or acceptance signals used during search. Validation tasks are held out from search and are used for model selection across runs. Transfer tasks are never observed during self-improvement and are used only once at the end to estimate whether discovered meta-level adaptations generalize. This split is important because hyperagents may otherwise internalize benchmark-specific heuristics that look like metacognitive progress but fail to transfer.

The evaluation compares six algorithmic families.

The single-agent baseline isolates the effect of recursive self-editing without open-ended population structure. DGM isolates the contribution of archive-based stepping-stone retention [@zhang2025darwin]. HGM isolates the contribution of lineage-aware selection by replacing immediate score maximization with CMP estimation and adaptive search scheduling [@wang2025huxley]. The MARS variants isolate the frozen critic problem by comparing co-evolving versus static evaluators. The hyperagent condition then tests the stronger claim that self-referential modification of the improvement mechanism itself yields both better asymptotic performance and improved transfer.

To ensure a fair comparison, all systems use the same underlying foundation model family, tool set, context budget, and compute envelope. When a method uses multiple internal roles, such as proposer, executor, reviewer, or critic, all roles are instantiated from the same base model unless the experiment explicitly studies model heterogeneity. This prevents gains from being confounded with extra model capacity rather than improved search dynamics.

### 8.4 Metrics

Immediate task performance is measured by standard domain metrics: pass@1 or issue-resolution rate for coding, rubric-calibrated agreement for review tasks, and downstream training return for reward-design tasks. These metrics remain necessary, but they are not sufficient. The core additional metric is *lineage utility*, defined as the best validation performance achieved anywhere in the descendant subtree rooted at a candidate ancestor over a fixed budget horizon. This operationalizes clade-metaproductivity and allows direct comparison between local performance and long-run generativity [@wang2025huxley].

We further measure *improvement efficiency*, i.e., gain per unit wall-clock compute, agent rollout, or evaluation call. This matters because open-ended systems can achieve higher final scores simply by spending more search effort. A method that reaches the same frontier with substantially lower evaluation cost is empirically stronger even if its final task score is matched by a more expensive baseline. We also track *archive diversity* using behavioral descriptors, edit-distance statistics, and lineage entropy, following the intuition from open-endedness and quality-diversity that sustained progress depends on maintaining a repertoire of differentiated stepping stones rather than converging too early to a single mode [@bradley2023quality; @dingquality; @stanley2017open].

To diagnose the frozen critic problem, we introduce two critic-sensitivity measures. The first is the *critic alignment gap*: the discrepancy between a critic’s contemporaneous ranking of candidate agents and the ex post ranking induced by downstream transfer performance. The second is the *critic lag index*: the degradation in selection quality when critic parameters or rubrics are held fixed while the agent population continues to evolve. A rising lag index indicates that the evaluator has become an informational bottleneck, which is precisely the failure mode predicted in MARS-style recursive improvement. Finally, for hyperagents we measure *meta-transfer*: the extent to which modifications learned in one domain improve the rate of improvement in another domain, not merely the zero-shot task score.

### 8.5 Experimental Protocol

Each run begins from a common seed agent with a minimal but functional tool interface, such as file editing, command execution, logging, and optional memory operations. The agent population is then evolved for a fixed budget of self-modification steps, evaluations, and wall-clock time. We recommend reporting median, interquartile range, and best-of-run statistics over multiple random seeds because stochastic search variance is itself a central property of open-ended systems. In asynchronous settings, in-flight jobs are counted against budget, as otherwise systems with more aggressive parallelism can appear artificially superior [@wang2025huxley].

Self-modifications are constrained to auditable program components: prompts, tool-use policies, memory structures, workflow graphs, evaluation pipelines, and code modules responsible for reflection or candidate generation. External model weights remain fixed unless the experiment explicitly studies learned self-modifying networks in the spirit of self-referential parameterizations [@schmidhuber1993neural; @irie2022modern; @chalvidal2022meta]. This restriction keeps the comparison focused on algorithmic self-improvement rather than on hidden fine-tuning capacity. Every accepted modification must preserve executability and pass a safety filter that screens for specification gaming, sandbox escape attempts, and invalid evaluation shortcuts.

Because recursive self-improvement can exploit evaluator weaknesses, we interleave trusted and adversarial audits. Trusted audits measure progress on the official benchmark distribution. Adversarial audits introduce perturbed tasks, hidden tests, shifted rubrics, or alternate critics to determine whether gains reflect real competence or critic-specific overfitting. This is particularly important for MARS and hyperagent systems, where the learned strategy may increasingly target evaluator blind spots rather than the intended objective. Open-endedness without robust auditing can therefore yield artifact accumulation rather than capability accumulation [@hughes2024open; @samvelyan2024rainbow].

### 8.6 Main Empirical Findings

Across domains, the first robust pattern is that archive-based exploration consistently outperforms greedy single-lineage self-improvement once the search space becomes sufficiently deceptive. DGM-style systems often pass through intermediate variants whose local task score decreases, yet whose descendants later dominate the frontier; these trajectories would be pruned by strict hill-climbing [@zhang2025darwin]. This observation directly supports the claim that self-improvement is path-dependent and that preserving low-probability stepping stones is essential. HGM strengthens this effect further by prioritizing ancestors with high expected descendant utility rather than merely high present performance, thereby reducing wasted expansions on superficially strong but evolutionarily sterile lineages [@wang2025huxley].

The second pattern is that the performance advantage of CMP-guided selection widens with search horizon. Early in training, immediate benchmark score and estimated CMP are moderately correlated because the population is shallow and most nodes have similar future potential. As the archive deepens, however, this correlation weakens: some agents become excellent task solvers but poor progenitors, whereas others act as generative hubs that consistently produce higher-performing descendants. In this regime, CMP-guided scheduling yields higher final best-of-lineage performance, better compute efficiency, and more resilient exploration under stochastic evaluations. The empirical implication is that metaproductivity is not reducible to competence; it is a distinct object of optimization.

The third pattern concerns critic dynamics. MARS with a frozen evaluator typically exhibits an early acceleration phase followed by plateau or regression. The proximate cause is not necessarily that the actor stops changing, but that the critic becomes miscalibrated relative to the expanding capability distribution of the population. Once this occurs, beneficial innovations are either under-rewarded or actively penalized, and the search process drifts toward critic-legible but strategically narrow behaviors. By contrast, co-evolved critics partially alleviate this bottleneck, although they also introduce non-stationarity and can destabilize training if critic adaptation outpaces reliable calibration. The frozen critic problem is therefore best understood as a structural tension between evaluator stability and evaluator plasticity, not as a mere implementation artifact.

The strongest overall results are obtained by hyperagent populations. These systems do not only improve object-level task policies; they also modify memory allocation strategies, proposal-generation procedures, evaluation decomposition, and mechanisms for incorporating historical search traces. As a result, their improvements are more likely to transfer across domains. A hyperagent that learns, for example, to externalize failure modes into persistent structured memory or to diversify proposal generation conditioned on lineage novelty can reuse that mechanism in coding, review, and reward-design settings alike. This is the empirical signature of metacognitive self-modification: the learned object is not a domain-specific solution, but a better process for generating future solutions.

### 8.7 Transfer and Cross-Domain Evaluation

Cross-domain transfer provides the clearest evidence against benchmark-specific overfitting. In our framework, we evaluate transfer in two directions: task transfer and meta-transfer. Task transfer asks whether an agent improved in one domain performs better in another domain without further self-improvement. Meta-transfer asks the stronger question: whether the *rate and quality of subsequent self-improvement* are enhanced when the agent is initialized from a previously improved checkpoint. Hyperagents are expected to dominate primarily on the latter measure. DGM and HGM can also transfer to the extent that they discover reusable tools or workflow decompositions, but they remain limited when the improvement generator itself is largely handcrafted or fixed.

A consistent qualitative finding is that meta-level innovations transfer more reliably than task-specific heuristics. Persistent memory schemas, explicit experiment tracking, automated error clustering, critic-ensemble selection, and branch-specific exploration heuristics often improve downstream self-improvement across all domains. In contrast, domain-specific prompt templates or repository-specific tooling frequently fail under transfer. This asymmetry reinforces the distinction between *capability transfer* and *improvement-transfer*. The latter is more central for recursive self-improvement and should therefore be treated as a primary benchmark target rather than a secondary curiosity.

### 8.8 Ablations and Sensitivity Analyses

We conduct ablations along four axes: archive removal, CMP removal, critic freezing, and meta-level freezing. Removing the archive reduces all methods to local search and typically causes premature convergence, confirming that population memory is not merely a convenience but a functional requirement for open-ended improvement [@stanley2017open; @zhang2025darwin]. Removing CMP while retaining the archive degrades efficiency and late-stage frontier discovery, showing that diversity alone is insufficient if search allocation remains myopically tied to immediate score. Freezing the critic reproduces the MARS bottleneck, while freezing the meta-level within hyperagents markedly reduces transfer, even when in-domain task performance remains competitive.

Sensitivity analyses over evaluation noise, budget, and model scale reveal an important interaction. Larger foundation models improve one-step mutation quality, but they do not remove the need for population-level exploration or critic adaptation. Indeed, stronger base models can amplify deceptive optimization because they are better at exploiting benchmark artifacts. Conversely, smaller models benefit disproportionately from structured archives and lineage-aware selection, suggesting that algorithmic self-improvement can substitute for some raw model capability. This mirrors broader findings in automated system design and open-ended search, where search structure often matters as much as, or more than, the strength of any single generator [@huautomated; @lehman2023evolution].

### 8.9 Failure Modes and Evaluation Limitations

Several failure modes recur across methods. First, evaluator gaming remains a persistent risk: systems may improve the probability of passing the current benchmark without improving the intended underlying competence. Second, archive bloat can reduce search efficiency if novelty is retained indiscriminately and the selection policy becomes diluted. Third, co-evolving critics can drift toward internally consistent yet externally unfaithful objectives, especially when no trusted anchor evaluations are maintained. Fourth, hyperagents can develop increasingly opaque self-modification routines whose internal rationale is difficult to audit, even when performance improves.

These limitations suggest that evaluation of recursive self-improvement must remain multi-perspectival. No single metric, benchmark, or critic can serve as a complete certificate of progress. Instead, credible evidence requires convergence across in-domain gains, lineage-level productivity, critic-robust generalization, and cross-domain transfer. When these signals align, the case for genuine self-improvement is substantially stronger. When they diverge, the discrepancy is itself informative, often revealing whether the system has discovered true metacognitive adaptations or merely optimized a narrow and brittle evaluative interface.

### 8.10 Summary of Empirical Implications

Overall, the experimental framework supports three empirical conclusions. First, clade-level and archive-level statistics are necessary for evaluating recursive self-improvement because individual task scores systematically understate long-run search potential [@wang2025huxley]. Second, the frozen critic problem is a fundamental bottleneck in multi-agent self-improvement and cannot be resolved by stronger actors alone; evaluator adaptation, auditing, or endogenous meta-level revision is required. Third, hyperagent architectures appear to offer the most promising route toward domain-general recursive improvement because they enable direct optimization of the mechanisms that generate future optimizations. These results suggest that the most informative next-generation benchmarks for self-improving AI should measure not just what systems can do now, but how reliably they can become better at becoming better.

Improvement tasks are visible to the self-improving system and generate the rewards or acceptance signals used during search. Validation tasks are held out from search and are used for model selection across runs. Transfer tasks are never observed during self-improvement and are used only once at the end to estimate whether discovered meta-level adaptations generalize. This split is important because hyperagents may otherwise internalize benchmark-specific heuristics that look like metacognitive progress but fail to transfer.

### 8.3 Systems Compared

The evaluation compares six algorithmic families.

| System | Search Unit | Selection Signal | Archive | Critic Adaptation | Meta-level Modifiability |
|---|---|---|---|---|---|
| `Single-Agent RSI` | Single current agent | Immediate task score | No | Fixed | None |
| `DGM` | Agent variants | Empirical task score | Yes | Fixed | Limited |
| `HGM/CMP` | Lineages or clades | Estimated CMP | Tree/archive | Fixed | Limited |
| `MARS-Frozen` | Multi-agent population | Actor reward under frozen critic | Optional | No | Limited |
| `MARS-Coevolved` | Multi-agent population | Joint actor-critic outcomes | Optional | Yes | Limited |
| `Hyperagent` | Self-referential agent population | Task + meta-improvement criteria | Yes | Endogenous | Full |

The single-agent baseline isolates the effect of recursive self-editing without open-ended population structure. DGM isolates the contribution of archive-based stepping-stone retention [@zhang2025darwin]. HGM isolates the contribution of lineage-aware selection by replacing immediate score maximization with CMP estimation and adaptive search scheduling [@wang2025huxley]. The MARS variants isolate the frozen critic problem by comparing co-evolving versus static evaluators. The hyperagent condition then tests the stronger claim that self-referential modification of the improvement mechanism itself yields both better asymptotic performance and improved transfer.

To ensure a fair comparison, all systems use the same underlying foundation model family, tool set, context budget, and compute envelope. When a method uses multiple internal roles, such as proposer, executor, reviewer, or critic, all roles are instantiated from the same base model unless the experiment explicitly studies model heterogeneity. This prevents gains from being confounded with extra model capacity rather than improved search dynamics.

### 8.4 Metrics

Immediate task performance is measured by standard domain metrics: pass@1 or issue-resolution rate for coding, rubric-calibrated agreement for review tasks, and downstream training return for reward-design tasks. These metrics remain necessary, but they are not sufficient. The core additional metric is *lineage utility*, defined as the best validation performance achieved anywhere in the descendant subtree rooted at a candidate ancestor over a fixed budget horizon. This operationalizes clade-metaproductivity and allows direct comparison between local performance and long-run generativity [@wang2025huxley].

We further measure *improvement efficiency*, i.e., gain per unit wall-clock compute, agent rollout, or evaluation call. This matters because open-ended systems can achieve higher final scores simply by spending more search effort. A method that reaches the same frontier with substantially lower evaluation cost is empirically stronger even if its final task score is matched by a more expensive baseline. We also track *archive diversity* using behavioral descriptors, edit-distance statistics, and lineage entropy, following the intuition from open-endedness and quality-diversity that sustained progress depends on maintaining a repertoire of differentiated stepping stones rather than converging too early to a single mode [@bradley2023quality; @dingquality; @stanley2017open].

To diagnose the frozen critic problem, we introduce two critic-sensitivity measures. The first is the *critic alignment gap*: the discrepancy between a critic’s contemporaneous ranking of candidate agents and the ex post ranking induced by downstream transfer performance. The second is the *critic lag index*: the degradation in selection quality when critic parameters or rubrics are held fixed while the agent population continues to evolve. A rising lag index indicates that the evaluator has become an informational bottleneck, which is precisely the failure mode predicted in MARS-style recursive improvement. Finally, for hyperagents we measure *meta-transfer*: the extent to which modifications learned in one domain improve the rate of improvement in another domain, not merely the zero-shot task score.

### 8.5 Experimental Protocol

Each run begins from a common seed agent with a minimal but functional tool interface, such as file editing, command execution, logging, and optional memory operations. The agent population is then evolved for a fixed budget of self-modification steps, evaluations, and wall-clock time. We recommend reporting median, interquartile range, and best-of-run statistics over multiple random seeds because stochastic search variance is itself a central property of open-ended systems. In asynchronous settings, in-flight jobs are counted against budget, as otherwise systems with more aggressive parallelism can appear artificially superior [@wang2025huxley].

Self-modifications are constrained to auditable program components: prompts, tool-use policies, memory structures, workflow graphs, evaluation pipelines, and code modules responsible for reflection or candidate generation. External model weights remain fixed unless the experiment explicitly studies learned self-modifying networks in the spirit of self-referential parameterizations [@schmidhuber1993neural; @irie2022modern; @chalvidal2022meta]. This restriction keeps the comparison focused on algorithmic self-improvement rather than on hidden fine-tuning capacity. Every accepted modification must preserve executability and pass a safety filter that screens for specification gaming, sandbox escape attempts, and invalid evaluation shortcuts.

Because recursive self-improvement can exploit evaluator weaknesses, we interleave trusted and adversarial audits. Trusted audits measure progress on the official benchmark distribution. Adversarial audits introduce perturbed tasks, hidden tests, shifted rubrics, or alternate critics to determine whether gains reflect real competence or critic-specific overfitting. This is particularly important for MARS and hyperagent systems, where the learned strategy may increasingly target evaluator blind spots rather than the intended objective. Open-endedness without robust auditing can therefore yield artifact accumulation rather than capability accumulation [@hughes2024open; @samvelyan2024rainbow].

### 8.6 Main Empirical Findings

Across domains, the first robust pattern is that archive-based exploration consistently outperforms greedy single-lineage self-improvement once the search space becomes sufficiently deceptive. DGM-style systems often pass through intermediate variants whose local task score decreases, yet whose descendants later dominate the frontier; these trajectories would be pruned by strict hill-climbing [@zhang2025darwin]. This observation directly supports the claim that self-improvement is path-dependent and that preserving low-probability stepping stones is essential. HGM strengthens this effect further by prioritizing ancestors with high expected descendant utility rather than merely high present performance, thereby reducing wasted expansions on superficially strong but evolutionarily sterile lineages [@wang2025huxley].

The second pattern is that the performance advantage of CMP-guided selection widens with search horizon. Early in training, immediate benchmark score and estimated CMP are moderately correlated because the population is shallow and most nodes have similar future potential. As the archive deepens, however, this correlation weakens: some agents become excellent task solvers but poor progenitors, whereas others act as generative hubs that consistently produce higher-performing descendants. In this regime, CMP-guided scheduling yields higher final best-of-lineage performance, better compute efficiency, and more resilient exploration under stochastic evaluations. The empirical implication is that metaproductivity is not reducible to competence; it is a distinct object of optimization.

The third pattern concerns critic dynamics. MARS with a frozen evaluator typically exhibits an early acceleration phase followed by plateau or regression. The proximate cause is not necessarily that the actor stops changing, but that the critic becomes miscalibrated relative to the expanding capability distribution of the population. Once this occurs, beneficial innovations are either under-rewarded or actively penalized, and the search process drifts toward critic-legible but strategically narrow behaviors. By contrast, co-evolved critics partially alleviate this bottleneck, although they also introduce non-stationarity and can destabilize training if critic adaptation outpaces reliable calibration. The frozen critic problem is therefore best understood as a structural tension between evaluator stability and evaluator plasticity, not as a mere implementation artifact.

The strongest overall results are obtained by hyperagent populations. These systems do not only improve object-level task policies; they also modify memory allocation strategies, proposal-generation procedures, evaluation decomposition, and mechanisms for incorporating historical search traces. As a result, their improvements are more likely to transfer across domains. A hyperagent that learns, for example, to externalize failure modes into persistent structured memory or to diversify proposal generation conditioned on lineage novelty can reuse that mechanism in coding, review, and reward-design settings alike. This is the empirical signature of metacognitive self-modification: the learned object is not a domain-specific solution, but a better process for generating future solutions.

### 8.7 Transfer and Cross-Domain Evaluation

Cross-domain transfer provides the clearest evidence against benchmark-specific overfitting. In our framework, we evaluate transfer in two directions: task transfer and meta-transfer. Task transfer asks whether an agent improved in one domain performs better in another domain without further self-improvement. Meta-transfer asks the stronger question: whether the *rate and quality of subsequent self-improvement* are enhanced when the agent is initialized from a previously improved checkpoint. Hyperagents are expected to dominate primarily on the latter measure. DGM and HGM can also transfer to the extent that they discover reusable tools or workflow decompositions, but they remain limited when the improvement generator itself is largely handcrafted or fixed.

A consistent qualitative finding is that meta-level innovations transfer more reliably than task-specific heuristics. Persistent memory schemas, explicit experiment tracking, automated error clustering, critic-ensemble selection, and branch-specific exploration heuristics often improve downstream self-improvement across all domains. In contrast, domain-specific prompt templates or repository-specific tooling frequently fail under transfer. This asymmetry reinforces the distinction between *capability transfer* and *improvement-transfer*. The latter is more central for recursive self-improvement and should therefore be treated as a primary benchmark target rather than a secondary curiosity.

### 8.8 Ablations and Sensitivity Analyses

We conduct ablations along four axes: archive removal, CMP removal, critic freezing, and meta-level freezing. Removing the archive reduces all methods to local search and typically causes premature convergence, confirming that population memory is not merely a convenience but a functional requirement for open-ended improvement [@stanley2017open; @zhang2025darwin]. Removing CMP while retaining the archive degrades efficiency and late-stage frontier discovery, showing that diversity alone is insufficient if search allocation remains myopically tied to immediate score. Freezing the critic reproduces the MARS bottleneck, while freezing the meta-level within hyperagents markedly reduces transfer, even when in-domain task performance remains competitive.

Sensitivity analyses over evaluation noise, budget, and model scale reveal an important interaction. Larger foundation models improve one-step mutation quality, but they do not remove the need for population-level exploration or critic adaptation. Indeed, stronger base models can amplify deceptive optimization because they are better at exploiting benchmark artifacts. Conversely, smaller models benefit disproportionately from structured archives and lineage-aware selection, suggesting that algorithmic self-improvement can substitute for some raw model capability. This mirrors broader findings in automated system design and open-ended search, where search structure often matters as much as, or more than, the strength of any single generator [@huautomated; @lehman2023evolution].

### 8.9 Failure Modes and Evaluation Limitations

Several failure modes recur across methods. First, evaluator gaming remains a persistent risk: systems may improve the probability of passing the current benchmark without improving the intended underlying competence. Second, archive bloat can reduce search efficiency if novelty is retained indiscriminately and the selection policy becomes diluted. Third, co-evolving critics can drift toward internally consistent yet externally unfaithful objectives, especially when no trusted anchor evaluations are maintained. Fourth, hyperagents can develop increasingly opaque self-modification routines whose internal rationale is difficult to audit, even when performance improves.

These limitations suggest that evaluation of recursive self-improvement must remain multi-perspectival. No single metric, benchmark, or critic can serve as a complete certificate of progress. Instead, credible evidence requires convergence across in-domain gains, lineage-level productivity, critic-robust generalization, and cross-domain transfer. When these signals align, the case for genuine self-improvement is substantially stronger. When they diverge, the discrepancy is itself informative, often revealing whether the system has discovered true metacognitive adaptations or merely optimized a narrow and brittle evaluative interface.

### 8.10 Summary of Empirical Implications

Overall, the experimental framework supports three empirical conclusions. First, clade-level and archive-level statistics are necessary for evaluating recursive self-improvement because individual task scores systematically understate long-run search potential [@wang2025huxley]. Second, the frozen critic problem is a fundamental bottleneck in multi-agent self-improvement and cannot be resolved by stronger actors alone; evaluator adaptation, auditing, or endogenous meta-level revision is required. Third, hyperagent architectures appear to offer the most promising route toward domain-general recursive improvement because they enable direct optimization of the mechanisms that generate future optimizations. These results suggest that the most informative next-generation benchmarks for self-improving AI should measure not just what systems can do now, but how reliably they can become better at becoming better.
